Every exercise in Digi Rehab can be tagged across five independent category dimensions. These tags drive filtering in the UI and allow providers to narrow down exercises by body region, movement pattern, required equipment, and more.

Each dimension is stored in its own database table and linked to exercises through a dedicated M:N association table. An exercise may have **zero or more** values from each dimension.

## The `is_custom` Flag

Every category row carries an `is_custom: bool` field.

- **Default categories** are seeded during `init_database.py` with `is_custom=False`. These entries represent the clinical standard taxonomy and **cannot be deleted** through the API.
- **Custom categories** are any entries created by API consumers (`POST /api/v1/categories/{category-type}`). These are created with `is_custom=True` and **can be updated or deleted** at any time.

<Warning>
Attempting to `DELETE` a category row where `is_custom=False` will be rejected by the API. Update your client to check the `is_custom` field before rendering a delete control.
</Warning>

## Retrieving Categories via API

All five category types share the same endpoint pattern under `/api/v1/categories/`.

```bash
# Body areas
GET /api/v1/categories/body-areas

# Exercise types
GET /api/v1/categories/exercise-types

# Positions
GET /api/v1/categories/positions

# Movements
GET /api/v1/categories/movements

# Equipment types
GET /api/v1/categories/equipment-types
```

Each endpoint returns a list of category objects:

```json
[
  {
    "id": 1,
    "name": "Stretch",
    "is_custom": false,
    "created_at": "2024-01-15T09:00:00Z"
  },
  {
    "id": 14,
    "name": "Proprioception",
    "is_custom": true,
    "created_at": "2024-03-10T14:22:00Z"
  }
]
```

## Creating a Custom Category

```bash
curl -X POST "http://localhost:8000/api/v1/categories/body-areas" \
  -H "Content-Type: application/json" \
  -d '{"name": "Temporomandibular Joint"}'
```

The response includes the newly assigned `id` and confirms `is_custom: true`:

```json
{
  "id": 14,
  "name": "Temporomandibular Joint",
  "is_custom": true,
  "created_at": "2024-06-01T10:30:00Z"
}
```

## Assigning Categories to an Exercise

Category IDs are passed as arrays of integers when creating or updating an exercise. Each array maps to one of the five dimensions:

```json
{
  "name": "Shoulder Flexion with Resistance Band",
  "instructions": "Stand with band under feet, pull upward to shoulder height...",
  "repetitions": 10,
  "sets": 3,
  "difficulty_level": "intermediate",
  "body_area_ids": [10],
  "exercise_type_ids": [9],
  "position_ids": [1],
  "movement_ids": [1],
  "equipment_type_ids": [1]
}
```

The same arrays are accepted on `PUT /api/v1/exercises/{exercise_id}` to replace the existing category assignments.

<Tip>
Fetch all category lists once on page load and cache them client-side. Category lists change infrequently and do not need to be re-fetched on every exercise form render.
</Tip>
