The Digi Rehab API is a RESTful service built on **FastAPI**. All request and response bodies use **JSON**. The API is versioned under the `/api/v1` prefix.

## Base URL

```
http://localhost:8000
```

## Version prefix

All resource endpoints are scoped under:

```
http://localhost:8000/api/v1
```

## Interactive documentation

<Info>
FastAPI automatically generates interactive API documentation while the server is running.

- **Swagger UI** — [http://localhost:8000/docs](http://localhost:8000/docs)
- **ReDoc** — [http://localhost:8000/redoc](http://localhost:8000/redoc)
</Info>

## System endpoints

### GET /

Returns API metadata including the current version and links to documentation.

```bash
curl http://localhost:8000/
```

```json
{
  "message": "Welcome to Digi Rehab API",
  "version": "1.0.0",
  "description": "Exercise Prescription System for Rehabilitation",
  "docs": "/docs",
  "redoc": "/redoc",
  "status": "operational"
}
```

### GET /health

Health check endpoint that verifies database connectivity. Returns `503` if the database is unreachable.

```bash
curl http://localhost:8000/health
```

```json
{
  "status": "healthy",
  "database": "connected",
  "timestamp": "2026-02-16T00:00:00Z"
}
```

### GET /api/v1

Returns version information and a map of all resource endpoint paths.

```bash
curl http://localhost:8000/api/v1
```

---

## Pagination

List endpoints support offset-based pagination via two query parameters:

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `skip` | integer | `0` | Number of records to skip |
| `limit` | integer | `100` | Maximum number of records to return |

```bash
curl "http://localhost:8000/api/v1/joints/?skip=0&limit=20"
```

---

## Response format

All responses are JSON. Successful responses return the resource object or an array of resource objects. Delete operations return a `MessageResponse` object:

```json
{ "message": "Resource deleted successfully" }
```

---

## HTTP status codes

| Code | Meaning |
|------|---------|
| `200 OK` | Request succeeded. |
| `201 Created` | Resource created successfully (POST endpoints). |
| `400 Bad Request` | Invalid input — e.g. duplicate name, referenced foreign key does not exist, or attempt to delete a non-custom category. |
| `404 Not Found` | The requested resource does not exist. |
| `422 Unprocessable Entity` | Request body failed schema validation (FastAPI/Pydantic). |
| `503 Service Unavailable` | Database is unreachable (health check only). |

Error bodies follow this shape:

```json
{ "detail": "Human-readable error message" }
```
