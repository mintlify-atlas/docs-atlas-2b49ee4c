All configuration is driven by a `.env` file in the project root. Copy `.env.example` to `.env` and adjust the values before starting the server.

```bash
cp .env.example .env
```

## Environment variables

| Variable | Default | Description |
|---|---|---|
| `DATABASE_URL` | `sqlite:///./digi_rehab.db` | SQLAlchemy connection string |
| `DEBUG` | `True` | Enable debug mode |
| `LOG_LEVEL` | `info` | Uvicorn log level (`debug`, `info`, `warning`, `error`) |
| `HOST` | `0.0.0.0` | Interface the server binds to |
| `PORT` | `8000` | TCP port the server listens on |
| `ALLOWED_ORIGINS` | `*` | Comma-separated CORS allowed origins |
| `API_PREFIX` | `/api/v1` | URL prefix for all API routes |
| `API_TITLE` | `Digi Rehab API` | Title shown in Swagger UI |
| `API_VERSION` | `1.0.0` | API version string |

## Database configuration

Digi Rehab uses SQLAlchemy, so any database with a supported dialect works. The `DATABASE_URL` format follows the standard SQLAlchemy connection URL scheme.

<Tabs>
  <Tab title="SQLite (default)">
    No additional drivers required. The database file is created automatically at first startup.

    ```bash
    DATABASE_URL=sqlite:///./digi_rehab.db
    ```

    The path `./digi_rehab.db` is relative to the directory where you launch the server.

    <Note>
      SQLite is suitable for local development and single-user evaluation. It does not support concurrent writes, so use PostgreSQL or MySQL for multi-user or production deployments.
    </Note>
  </Tab>

  <Tab title="PostgreSQL">
    Install the driver first:

    ```bash
    pip install psycopg[binary]
    ```

    The `psycopg` (v3) package is already listed in `requirements.txt`.

    Then set your connection string:

    ```bash
    DATABASE_URL=postgresql://username:password@localhost:5432/digi_rehab
    ```

    Replace `username`, `password`, and the database name as appropriate. If your PostgreSQL server uses SSL, append `?sslmode=require`:

    ```bash
    DATABASE_URL=postgresql://username:password@db.example.com:5432/digi_rehab?sslmode=require
    ```
  </Tab>

  <Tab title="MySQL">
    Install the driver:

    ```bash
    pip install pymysql
    ```

    Set your connection string using the `mysql+pymysql` dialect:

    ```bash
    DATABASE_URL=mysql+pymysql://username:password@localhost:3306/digi_rehab
    ```

    Create the database manually before first startup:

    ```sql
    CREATE DATABASE digi_rehab CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
    ```
  </Tab>
</Tabs>

## CORS configuration

CORS is configured in `main.py` using FastAPI's `CORSMiddleware`:

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # Configure appropriately for production
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

<Warning>
  The default `allow_origins=["*"]` permits requests from any origin. This is intentional for development but **must be restricted before deploying to production**. Replace `["*"]` with an explicit list of your frontend origins, for example:

  ```python
  allow_origins=[
      "https://app.yourcompany.com",
      "https://staging.yourcompany.com",
  ]
  ```
</Warning>

The `ALLOWED_ORIGINS` variable in `.env` is available as a configuration reference but the middleware in `main.py` must be updated manually to read from it if you want environment-driven CORS control.

## API server configuration

The server is started by the `if __name__ == "__main__"` block in `main.py`, which calls `uvicorn.run` with the following defaults:

```python
uvicorn.run(
    "main:app",
    host="0.0.0.0",
    port=8000,
    reload=True,   # auto-reload on file changes — disable in production
    log_level="info"
)
```

To override host, port, or reload behaviour without editing `main.py`, launch uvicorn directly:

```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --no-reload --log-level warning
```

<Tip>
  Set `--no-reload` in production. The file-watcher used by `reload=True` adds overhead and is not safe in containerised environments where the filesystem may be read-only.
</Tip>

## Frontend environment

The React frontend (Vite) connects to the API at `http://localhost:8000` by default. If you change the API host or port, update the base URL in your frontend API client configuration.

To build the frontend for production:

```bash
cd frontend
npm run build
```

The compiled static assets are written to `frontend/dist/` and can be served by any static host or placed behind a reverse proxy alongside the FastAPI server.
