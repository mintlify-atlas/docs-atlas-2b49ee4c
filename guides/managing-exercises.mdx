title: "Managing Exercises"
description: "Build and organize the exercise library, including category associations and problem linkages."
---

Exercises are the core clinical content in Digi Rehab. Each exercise can be tagged with up to five category dimensions — body area, exercise type, position, movement, and equipment type — and linked to one or more clinical problems.

## Exercise fields

| Field | Type | Required | Notes |
|---|---|---|---|
| `name` | string | Yes | 1–150 characters |
| `description` | string | No | Brief summary |
| `instructions` | string | Yes | Step-by-step instructions |
| `duration_minutes` | integer | No | Must be > 0 |
| `repetitions` | integer | No | Must be > 0 |
| `sets` | integer | No | Must be > 0 |
| `difficulty_level` | string | No | `beginner`, `intermediate`, or `advanced` |
| `image_url` | string | No | URL to an image |
| `video_url` | string | No | URL to a video |
| `is_predefined` | boolean | No | `true` marks the exercise as a library entry (default `false`) |
| `repetition_suggestions` | string[] | No | Free-text ramp-up suggestions |
| `problem_ids` | integer[] | No | IDs of associated problems |
| `body_area_ids` | integer[] | No | IDs from `/api/v1/categories/body-areas` |
| `exercise_type_ids` | integer[] | No | IDs from `/api/v1/categories/exercise-types` |
| `position_ids` | integer[] | No | IDs from `/api/v1/categories/positions` |
| `movement_ids` | integer[] | No | IDs from `/api/v1/categories/movements` |
| `equipment_type_ids` | integer[] | No | IDs from `/api/v1/categories/equipment-types` |

<Note>
  All category IDs must reference records that already exist in the database. Use the [Custom Categories](/guides/custom-categories) guide to look up or add categories before creating exercises.
</Note>

## Endpoints

### Create an exercise

```bash
curl -X POST "http://localhost:8000/api/v1/exercises/" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Hamstring Stretch",
    "description": "Seated hamstring stretch targeting the posterior knee.",
    "instructions": "Sit on the floor with both legs extended. Hinge forward at the hips, reaching towards your feet. Hold for 30 seconds. Keep knees flat on the floor.",
    "duration_minutes": 2,
    "repetitions": 3,
    "sets": 1,
    "difficulty_level": "beginner",
    "is_predefined": true,
    "problem_ids": [1],
    "body_area_ids": [8],
    "exercise_type_ids": [1],
    "position_ids": [2],
    "movement_ids": [1],
    "equipment_type_ids": [12]
  }'
```

### List exercises

<CodeGroup>

```bash All exercises
curl "http://localhost:8000/api/v1/exercises/?skip=0&limit=50"
```

```bash Filter by difficulty
curl "http://localhost:8000/api/v1/exercises/?difficulty=beginner"
```

```bash Predefined library only
curl "http://localhost:8000/api/v1/exercises/predefined"
```

</CodeGroup>

### Get a single exercise

```bash
curl "http://localhost:8000/api/v1/exercises/5"
```

Returns the full exercise object including all nested category objects.

### Get exercises for a problem

```bash
curl "http://localhost:8000/api/v1/exercises/problem/1"
```

Returns all exercises linked to problem `1`. Returns `404 Not Found` if the problem does not exist.

### Update an exercise

Only the fields you supply are updated. Passing a category ID array **replaces** the entire set of associations for that category type.

```bash
curl -X PUT "http://localhost:8000/api/v1/exercises/5" \
  -H "Content-Type: application/json" \
  -d '{
    "difficulty_level": "intermediate",
    "repetitions": 5,
    "body_area_ids": [7, 8],
    "exercise_type_ids": [1, 3]
  }'
```

### Delete an exercise

```bash
curl -X DELETE "http://localhost:8000/api/v1/exercises/5"
```

```json
{ "message": "Exercise deleted successfully" }
```

### Search exercises

Searches across name, description, and instructions (case-insensitive, partial match).

```bash
curl "http://localhost:8000/api/v1/exercises/search/hamstring"
```

### Add or remove a problem association

<CodeGroup>

```bash Add problem
curl -X POST "http://localhost:8000/api/v1/exercises/5/problems/2"
```

```bash Remove problem
curl -X DELETE "http://localhost:8000/api/v1/exercises/5/problems/2"
```

</CodeGroup>

## Complete workflow: create an exercise with categories

<Steps>
  <Step title="Look up category IDs">
    Fetch the relevant category lists to find the IDs you need.

    ```bash
    curl "http://localhost:8000/api/v1/categories/body-areas"
    curl "http://localhost:8000/api/v1/categories/exercise-types"
    curl "http://localhost:8000/api/v1/categories/positions"
    ```
  </Step>

  <Step title="Create the exercise with category IDs">
    Supply the IDs collected in the previous step.

    ```bash
    curl -X POST "http://localhost:8000/api/v1/exercises/" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "Shoulder Flexion with Resistance Band",
        "description": "Strengthen shoulder flexors using elastic resistance.",
        "instructions": "Stand with the band anchored under both feet. Hold one end in each hand. Raise both arms forward to shoulder height with elbows straight. Lower slowly. Repeat.",
        "repetitions": 10,
        "sets": 3,
        "difficulty_level": "intermediate",
        "is_predefined": true,
        "body_area_ids": [10],
        "exercise_type_ids": [9],
        "position_ids": [1],
        "movement_ids": [1],
        "equipment_type_ids": [1]
      }'
    ```
  </Step>

  <Step title="Link the exercise to a problem">
    Either include `problem_ids` in the create request, or add them afterward.

    ```bash
    curl -X POST "http://localhost:8000/api/v1/exercises/6/problems/2"
    ```
  </Step>

  <Step title="Verify the full exercise record">
    ```bash
    curl "http://localhost:8000/api/v1/exercises/6"
    ```

    Confirm that `body_areas`, `exercise_types`, `positions`, `movements`, `equipment_types`, and `problems` all appear as expected in the response.
  </Step>
</Steps>

## Python example

The following snippet shows how to resolve category names to IDs programmatically before creating an exercise.

```python
import requests

BASE = "http://localhost:8000/api/v1"

# Resolve category IDs by name
body_areas = requests.get(f"{BASE}/categories/body-areas").json()
knee_id = next(ba["id"] for ba in body_areas if ba["name"] == "Knee")

exercise_types = requests.get(f"{BASE}/categories/exercise-types").json()
stretch_id = next(et["id"] for et in exercise_types if et["name"] == "Stretch")

# Create the exercise
exercise_data = {
    "name": "Hamstring Stretch",
    "instructions": "Sit on floor with legs extended, hinge forward at hips.",
    "difficulty_level": "beginner",
    "repetitions": 3,
    "duration_minutes": 2,
    "body_area_ids": [knee_id],
    "exercise_type_ids": [stretch_id],
    "position_ids": [2],  # Sitting
}

response = requests.post(f"{BASE}/exercises/", json=exercise_data)
print(response.json())
```
