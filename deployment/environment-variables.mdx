---
title: "Environment Variables"
description: "Complete reference for all environment variables used by the Digi Rehab API server."
---

Digi Rehab reads configuration from a `.env` file in the project root using `python-dotenv`. Copy `.env.example` to `.env` and adjust the values before starting the server.

<Warning>
  Never commit your `.env` file to version control. It may contain database credentials or secret keys. Add `.env` to your `.gitignore` file.
</Warning>

## Example .env File

This is the complete `.env.example` shipped with the project:

```bash .env.example
# Database Configuration
DATABASE_URL=sqlite:///./digi_rehab.db

# For PostgreSQL
# DATABASE_URL=postgresql://username:password@localhost:5432/digi_rehab

# For MySQL
# DATABASE_URL=mysql+pymysql://username:password@localhost:3306/digi_rehab

# Application Configuration
DEBUG=True
LOG_LEVEL=info

# Server Configuration
HOST=0.0.0.0
PORT=8000

# CORS Configuration (comma-separated origins)
ALLOWED_ORIGINS=*

# API Configuration
API_PREFIX=/api/v1
API_TITLE=Digi Rehab API
API_VERSION=1.0.0

# Security (for production)
# SECRET_KEY=your-secret-key-here
# ACCESS_TOKEN_EXPIRE_MINUTES=30
```

## Variable Reference

### Database

<ParamField query="DATABASE_URL" type="string" default="sqlite:///./digi_rehab.db">
  Full database connection string. SQLAlchemy uses this to determine both the database type and the connection parameters. Changing this value is the only step required to switch databases.

  ```bash SQLite (default)
  DATABASE_URL=sqlite:///./digi_rehab.db
  ```

  ```bash PostgreSQL
  DATABASE_URL=postgresql://username:password@localhost:5432/digi_rehab
  ```

  ```bash MySQL
  DATABASE_URL=mysql+pymysql://username:password@localhost:3306/digi_rehab
  ```
</ParamField>

### Application

<ParamField query="DEBUG" type="boolean" default="True">
  Enables debug mode. Set to `False` in production to suppress detailed error messages in API responses.
</ParamField>

<ParamField query="LOG_LEVEL" type="string" default="info">
  Logging verbosity level passed to the application. Accepted values follow Python's standard logging levels: `debug`, `info`, `warning`, `error`, `critical`.
</ParamField>

### Server

<ParamField query="HOST" type="string" default="0.0.0.0">
  The network interface the uvicorn server binds to. `0.0.0.0` makes the server accessible from any network interface. Use `127.0.0.1` to restrict to localhost only.
</ParamField>

<ParamField query="PORT" type="integer" default="8000">
  The TCP port the uvicorn server listens on.
</ParamField>

### CORS

<ParamField query="ALLOWED_ORIGINS" type="string" default="*">
  Comma-separated list of allowed CORS origins. The default `*` permits requests from any origin and is suitable for development only.
</ParamField>

<Note>
  CORS is configured in code inside `main.py`, not enforced directly from environment variables at the middleware level. The `CORSMiddleware` is currently hardcoded with `allow_origins=["*"]`, `allow_credentials=True`, `allow_methods=["*"]`, and `allow_headers=["*"]`. Update these values in `main.py` before deploying to production.

  ```python main.py
  app.add_middleware(
      CORSMiddleware,
      allow_origins=["*"],  # Configure appropriately for production
      allow_credentials=True,
      allow_methods=["*"],
      allow_headers=["*"],
  )
  ```
</Note>

### API

<ParamField query="API_PREFIX" type="string" default="/api/v1">
  URL prefix applied to all API routers. All resource endpoints (joints, problems, exercises, prescriptions, categories) are registered under this prefix.
</ParamField>

<ParamField query="API_TITLE" type="string" default="Digi Rehab API">
  Title string displayed in the Swagger UI (`/docs`) and ReDoc (`/redoc`) documentation pages.
</ParamField>

<ParamField query="API_VERSION" type="string" default="1.0.0">
  Version string displayed in the API documentation.
</ParamField>

### Security (production)

<ParamField query="SECRET_KEY" type="string">
  Secret key used for cryptographic signing. Commented out in `.env.example`. Required if you add JWT authentication or session management to the application.
</ParamField>

<ParamField query="ACCESS_TOKEN_EXPIRE_MINUTES" type="integer" default="30">
  Lifetime in minutes of access tokens. Commented out in `.env.example`. Used if JWT authentication is implemented.
</ParamField>
